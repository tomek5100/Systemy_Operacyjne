Zadanie 1. drzewko1.c, drzewko2.c

Zmodyfikować program procesy.c z poprzedniego zestawu tak, aby generował poniższe drzewa genealogiczne procesów. Należy w odpowiednim momencie przerwać pętlę poprzez dodanie jednej linii do programu. Aby nie tworzyć procesów zombi/osieroconych procesy muszą czekać na procesy potomne wywołując odpowiednią liczbę razy funkcję wait() tuż przed końcem funkcji main (za pętlą).

Uwaga: Nie należy zagnieżdżać instrukcji switch. Oba programy mają być jedynie drobną modyfikacją programu procesy.c (różniące się jedną linią kodu).

Wskazówka: Wystarczy nieznacznie zmodyfikować program procesy.c tak aby przerwać pętlę for w odpowiednim miejscu. Ponieważ procesy mają różną liczbę procesów potomnych (0-3), a zatem wywołują wait różną liczbę razy, należy wprowadzić licznik procesów potomnych.

Zadanie 2. macierzysty.c, potomny.c

Zmodyfikować program procesy.c tak, aby komunikaty procesów potomnych były wypisywane przez program uruchamiany funkcją execv().

Program potomny.c zawiera jedynie funkcję main(), która tylko wywołuje procinfo() aby wypisać informacje o procesie. Program macierzysty.c wywołuje w pętli w procesach potomnych program potomny.x (for + fork + execv), dodatkowo na samym początku powinien wypisać informacje o procesie używając procinfo().

W obu przypadkach należy dołączyć plik nagłówkowy #include "procinfo.h"

Zadanie 3. Zombi i sieroty

  1. Napisać program zombi.c, który utworzy proces zombi.
  2. Napisać program orphan.c, który utworzy proces osierocony.
    Przy użyciu sleep (nie używając funkcji wait) sprawić by procesy potomne były adoptowane przez proces init lub systemd.
    Poleceniem pstree -p wyświetlić drzewo procesów w danym systemie i zidentyfikować proces adoptujący osierocone procesy.

W obu przypadkach należy odpowiednio użyć funkcji fork() i sleep().
Aby na bieżąco monitorować interesujące procesy można użyć komend watch i ps:

watch -n 1 "ps -HFC zombi.x"

  1. Napisać program execute.c, który:

    • Wypisze “pierwszy komunikat” używając funkcji printf(), sprawdzić co się stanie gdy pominiemy znak końca linii \n
    • Bez użycia funkcji fork() wywoła funkcję execl() aby wykonać program potomny.x
    • Wypisze “drugi komunikat”
